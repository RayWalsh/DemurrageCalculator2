Changelog

v1.3 – May 2025

  • Column Management Settings Panel: A new dedicated settings page at /settings/case-management allows viewing and managing SQL columns.
  • Add New Column: Users can add new columns to the Cases SQL table via a modal. Column types are user-friendly (e.g. "Text", "Decimal").
  • Dynamic Column Detection: New SQL columns are detected and automatically added to the column picker in Case Management (hidden by default).
  • Edit Column Details: Users can rename and change the friendly field type of existing columns via a modal editor in the settings panel.
  • Settings Modals Load On Demand: Modals for editing or adding columns are hidden by default and only appear when triggered.
Changed
  • Column Picker Logic: Local storage preferences now merge with real SQL column metadata on page load, ensuring the UI adapts to new fields.

v1.2 – April 2025

  • Clickable DeepBlueRef: Clicking the case ID opens the Edit Case modal.
  • Saved Column Preferences: User-selected column visibility is now saved across sessions using localStorage.
  • Reset View Button: Users can reset their column layout to the default via the column picker panel.
Changed
  • Removed VesselName Click: Only DeepBlueRef is now used for opening the Edit Case modal.
  • Visual Cleanup: Header buttons aligned for better layout; minor spacing and hover adjustments.

v1.1 – March 2025

Initial deployment of the Case Management interface and modal editor.
